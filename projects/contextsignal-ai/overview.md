# ContextSignal.ai

## Goal

Build a profitable content platform teaching professionals how to use AI in their specific roles. Target: $5k–$10k/month revenue by Month 12, with <50% dependence on Google organic traffic.

## Context

**What it is:** A context engineering content site organised by audience vertical (not tool category). Each vertical targets a professional role (PMs, Sales, Designers, etc.) with workflow guides, prompt libraries, and tool recommendations.

**Part of:** Signal Media Portfolio. Domain: contextsignal.ai

**Business model:** Affiliate commissions (30%), digital products (25%), newsletter sponsorships (20%), display ads (15%), community/consulting (10%).

**Content model — 3 tiers:**
- **Tier 1 (Automated):** Claude API + Make.com pipeline → drafts only, never auto-publish. News/updates. 5–10/week across all verticals.
- **Tier 2 (Editorial):** AI-drafted, human-edited (30–60 min). Workflow guides + comparisons. 2–3/week per vertical.
- **Tier 3 (Flagship):** Personally crafted (2–4 hrs). Brand-defining guides + frameworks. 1–2/month per vertical.

**8 audience verticals, launched in waves:**
- Wave 1 (Weeks 1–6): Project Managers + Sales Professionals
- Wave 2 (Weeks 7–12): Designers + Developers
- Wave 3 (Month 4+): Traders, Marketers, Writers, Founders

**Signature frameworks (brand moats):** AI PM OS, SDR Personalisation Stack, ContextSignal Score.

**Tech stack:** WordPress on Kinsta (~$35/mo), GeneratePress theme, Rank Math SEO, Make.com automation, Claude API, Beehiiv email, Gumroad/LemonSqueezy for products, Pretty Links for affiliates, GA4 + Search Console.

**Monthly cost:** ~$65–85 early stage, ~$130–200 growth stage.

**Time commitment:** 5–7 hrs/week first 12 weeks, dropping to 2–3 hrs/week by Month 7+.

**Key strategy doc:** `ContextSignal-Strategy-v5.2.2-Final.md` in this folder — full strategy with content calendars, editorial policy, compliance frameworks, and detailed specs.

## Decisions

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-03 | Pivot from tool reviews to audience-vertical model | Generic tool listicles too saturated; role-specific workflows differentiate and unlock digital products |
| 2026-03 | Draft-only automation (never auto-publish) | Protects quality and avoids Google penalties for thin/duplicate AI content |
| 2026-03 | Delay Traders vertical to Wave 3 | Compliance friction + ad-network sensitivity; needs framework built first |
| 2026-03 | Cap Tier 1 at 2 publishes/week during Weeks 2–6 | Avoid cannibalisation; prove quality before scaling volume |
| 2026-03 | Use Beehiiv for email | Free to 2,500 subs; custom sending domain support; built-in segmentation |
